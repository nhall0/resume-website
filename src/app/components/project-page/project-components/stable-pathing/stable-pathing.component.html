This project was originally intended to be much larger, with an entirely different scope. Its goal was to create a game that had unlimited content, fueled by the glorious advent of AI. The players environment would be parsed into a 'Context' object, which would then be used to generate the game world. If similar contexts from the past were found, a manager would reuse previously generated assets. Quickly, I realized I'd bitten off more than I could chew. Managing the context and then returning something usable to the player ended up being incredibly challenging. Thus, the project sat in stasis after creating the world generation pipeline.

I used a Stable Diffusion model to get the base images - see one to the right. Said model was trained on a dataset of DnD maps. The previously mentioned 'Context' would be fed into an RAG model such as ChatGPT, and would produce a prompt for the environment. If the player had just gone through a rusty gate into the side of a cliff - clearly the outcome would be something akin to a dungeon or cave. The Stable Diffusion model would then generate a map that would be used to create the game world.

The images were then processed through a series of filters. First, the image would be put through a threshhold to get high-contrast image. It would then be divided into quarters (or more), and set aside for re-generation. Each quarter is then sent back through the Stable Diffusion model - with descriptors of the target environment. Finally, the high-contrast images were used as a mask against the newly generated images to carve out a path the player could traverse.

I used Unity to mock out the design and core-features of the game. The map image itself was loaded as an overlay resource - then the mask was used as the collision map underneath. This was done in order to have clearly translated graphics, while allowing the player to have a clean, intuitive experience traversing the map. As seen in the demo to the left, shadows and lighting were used in similar manner. All terrain and objects were generated via the same prompt pipeline as the map was.

Needless to say, parts of this project could have been done quite a bit better. The biggest improvement would come from not using AI (blasphemy!) to generate the base maps. AI would only be used to render the veneer of the world. The base map could be generated with a conventional map generation algorithm - my choice would be some cellular automata variant.

Ultimately, this was a fun experiment rather than a feasible concept. Much of the process, while automated via code, was still done manually. ChatGPT 3.0 was used for prompt creation, and would often generate inaccurate prompts. The Stable Diffusion model I'd used would also encounter similar issues, generating unusable maps. If I'd been more proficient with training models, I likely could have gotten better results.
