This project was mainly inspired by 'Bethesda-style' dialogue systems. I wanted to make a tool that could be useful to write in such a style, all bells and whistles attached. The application itself is quite simple, you create a node by clicking on the button within the canvas, and start writing your dialogue. The node auto-populates new routes with the options you write, and you can create new nodes that connect to those routes. Dialogue can be exported on the fly, and imported back. There are also Godot + Unity integrations, which parse the dialogue into a format one can use to integrate into their game/application.

Incredibly simple, it's basic graph logic. The more explicit data-structure would be a digraph, where the degree of each vertex is the number of options leading away from that node + 1. Each node has a slew of metadata. Certain boolean flags can be set and checked upon either arriving at a node or selecting an options - this is in the case you wanted to make a reactive choice. I.E - you visit someone for the second time, or you have a certain item in your inventory. The boolean flag system also allows for flags to be set outside of the application itself, giving the end-user a port of entry for their own game logic.

Developed in Godot, using GDScript. For those who havent worked with it, GDScript is very similar to Python. For those who don't know what Godot is at all, it's an open-source game-engine. I chose it because I thought it'd be cool to write a tool in the same language I'd be using to integrate it into a game - very much inspired by the engines own origins (Godot is actually coded in Godot). Exports are in JSON - chances are either in the future, or next time I do something similar, I'll try and write my own format. Check out the codebase in the link above.
